Data tools:

            This section was the most difficult. At the same time I learned a lot about how data is stored in spreadsheets such as text files and CSV (comma-separated files). Databases are another way data is stored through a computer. In order for the programmers to interact with the databases they need to use a query language called SQL (Structured Query Language). With SQL you will be able to make graphs and commands to fill out the spreadsheet. When using SQL there are certain functions that help create different outputs. The COUNTA for spreadsheet COUNT function for SQL tell you how many rows are on a side you choose to count. The AVERAGE function for spreadsheet and AVG function for SQL helps average the data in a desired column. MIN/MAX functions help you see the minimum and maximum of a data set. The SUM function tells you how much of everything you have in total. These functions will help you analyze and find trends in your data. Graphs are a great way to keep track of trends. There are also charts that keep track of fluctuations on your data and also charts that have different data that’s related that has correlated data. With all this data in line you’ll be able to make predicting based on the algorithms as well.

 

Big data:

            This part was my mastery section. I was able to learn a lot and retain the information. I learned that every aspect of internet usage and IoT usage is tracked and put into data sets. The data world is massive. Because the amount information needed to be stored is exponentially growing, by the year 2025 will have 175 zettabytes of stored data. One example of a big data source is the Large Hadron Collider which is the world largest particle accelerator and produces 50-75 million petabytes per year of data and NASA EOSDIS is a group that collects satellite footage and creates 23 terabytes per day. Digital libraries which hold a vast collection of historical documents, and media components uses a lot of data storage. Medical record such as CT scans and X-rays can collect a lot of data for each patient. Since the scans are divided in slides, they take up a lot of storage. The CT scan of the brain can take up to 16 MB. Since hospitals have a lot of patients, their databases need be large in order to keep all of their patients data stored. The data is stored for massive companies are stored in data centers. These have rows with racks of hard drives that have the data stored. They require lots of maintenance and work due to the vast amount of wiring involved and cooling systems. Large data sets can take a lot of time to process so engineers have created a way to speed it up. One way is parallel computing. This is when each piece of data is distributed among other machines that work on the same set of data.
